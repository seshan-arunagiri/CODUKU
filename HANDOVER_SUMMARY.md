## 1. Project Overview
CODUKU is a full-stack competitive coding platform (LeetCode/HackerRank clone) with a Harry Potter theme. Users are sorted into houses (Gryffindor, Slytherin, etc.) and compete on leaderboards by solving coding problems in 18+ languages.

## 2. Technical Stack
- **Frontend**: React 18, TypeScript, Monaco Editor, Zustand (State), Framer Motion (Animations).
- **Backend**: FastAPI (Python), Uvicorn (ASGI), Pydantic (Validation).
- **Database**: 
  - **PostgreSQL**: Primary data (Users, Problems, Leaderboards).
  - **Redis**: Real-time leaderboards and caching.
  - **In-Memory**: `SUBMISSIONS_DB` (currently holds active session submissions in the Judge service).
- **Infrastructure**: Docker, Docker Compose, NGINX (API Gateway).
- **Execution Engine**: Judge0 (Sandboxed code execution).

## 3. Architecture & Routing
The system uses a Microservices architecture behind an NGINX Gateway (Port 80/443).

| Service | Port | Endpoint Prefix | Purpose |
|---------|------|-----------------|---------|
| **Gateway** | 80 | `/api/v1/` | NGINX Reverse Proxy |
| **Auth** | 8001 | `/api/v1/auth` | JWT Auth, User Mgmt |
| **Judge** | 8002 | `/api/v1/submissions` | Code Execution, Problem MGMT |
| **Leaderboard**| 8003 | `/api/v1/leaderboard` | Real-time rankings (Redis) |
| **Mentor** | 8004 | `/api/v1/mentor` | AI Hints (OpenAI integrated) |
| **Judge0** | 2358 | - | Remote Code Sandbox |

## 4. Key Workflows & Data Flow

### Submission Flow:
1. **Frontend**: Code + ProblemID + UserInfo → `POST /api/v1/submissions`.
2. **Judge Service**: Saves to `SUBMISSIONS_DB` → Submits to **Judge0 API**.
3. **Judge0**: Compiles & Executes → Returns Result.
4. **Judge Service**: Maps Verdict (Accepted, WA, TLE) → Updates `SUBMISSIONS_DB`.
5. **Leaderboard**: If "Accepted", background task updates Redis/Postgres scores.
6. **Frontend**: Polls/Receives result → Confetti 🎊 + Score Update.
