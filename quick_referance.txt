# Quick Reference & Deployment Checklist

## 🚀 Quick Start Commands

### Local Development (30 seconds setup)

```bash
# Terminal 1: Backend
cd backend
python -m venv venv
source venv/bin/activate  # or `venv\Scripts\activate` on Windows
pip install -r requirements.txt
python app.py

# Terminal 2: Frontend
cd frontend
npm install
npm start

# Terminal 3: MongoDB (if not running)
mongod
```

### Docker Setup (1 command)

```bash
docker-compose up
```

## 📊 API Endpoints Reference

### Authentication
```
POST   /api/auth/register        # Create new account
POST   /api/auth/login           # Login (returns JWT)
```

### Questions
```
GET    /api/questions            # List all questions
GET    /api/questions/{id}       # Get specific question
POST   /api/admin/questions      # Create question (admin)
PUT    /api/admin/questions/{id} # Edit question (admin)
```

### Submissions
```
POST   /api/submit               # Submit code & get score
```

### Leaderboards
```
GET    /api/leaderboards/global  # Top 100 students
GET    /api/leaderboards/houses  # House rankings
GET    /api/leaderboards/house/{name}  # House members
```

### User Profile
```
GET    /api/user/profile         # Current user info
GET    /api/user/submissions     # User's submissions
```

### Houses
```
GET    /api/houses               # All house info
```

## 📈 Deployment Checklist

### Pre-Deployment ✅
- [ ] Test all features locally
- [ ] Change JWT_SECRET_KEY
- [ ] Update MONGO_URI to production database
- [ ] Enable HTTPS
- [ ] Set CORS origins
- [ ] Update API_URL in frontend .env
- [ ] Run security audit
- [ ] Load test the application
- [ ] Setup monitoring & logging
- [ ] Create database backups

### Backend Deployment Options

#### Heroku
```bash
heroku create your-app
heroku config:set MONGO_URI=your_prod_uri
heroku config:set JWT_SECRET_KEY=your_prod_key
git push heroku main
```

#### AWS (EC2)
```bash
# SSH into instance
ssh -i key.pem ec2-user@your-instance

# Install dependencies
sudo yum install python3 pip nodejs git mongodb
git clone your-repo
cd coding-platform
pip install -r requirements.txt
gunicorn -w 4 app:app
```

#### Google Cloud / Azure
Follow similar steps with their respective CLI tools.

### Frontend Deployment Options

#### Vercel (Recommended)
```bash
npm install -g vercel
vercel
# Follow prompts
vercel env add REACT_APP_API_URL
```

#### Netlify
```bash
npm run build
netlify deploy --prod --dir=build
```

#### AWS S3 + CloudFront
```bash
npm run build
aws s3 sync build/ s3://your-bucket
```

## 🔍 Monitoring & Maintenance

### Health Checks
```bash
# Backend health
curl http://localhost:5000/api/health

# Database connection test
python -c "from pymongo import MongoClient; print(MongoClient('mongodb://localhost:27017/').list_database_names())"

# Frontend running
curl http://localhost:3000
```

### Common Maintenance Tasks

#### Backup Database
```bash
# Backup
mongodump --out ./backup

# Restore
mongorestore ./backup

# Atlas backup (automatic, weekly)
# Portal: https://cloud.mongodb.com > Backups
```

#### Monitor Submissions
```
GET /api/admin/submissions  # Future endpoint to add
```

#### Clear Old Submissions (Monthly)
```javascript
db.submissions.deleteMany({
  submitted_at: {
    $lt: new Date(new Date().setFullYear(new Date().getFullYear() - 1))
  }
})
```

## 🐛 Common Issues & Solutions

| Issue | Solution |
|-------|----------|
| CORS Error | Check backend CORS config, ensure frontend API URL is correct |
| Port 5000/3000 in use | Kill process: `lsof -i :5000` then `kill -9 PID` |
| MongoDB connection fails | Ensure MongoDB is running: `mongod` or `brew services start mongodb-community` |
| JWT token expired | Clear localStorage, log back in |
| Code execution fails | Check Python syntax, ensure test cases are valid JSON |
| Slow leaderboard | Add database indexes, implement caching |
| Out of memory | Increase Docker memory limits or server RAM |

## 🎯 Development Workflow

### Adding a New Feature

1. **Design Phase**
   - Plan database changes in ARCHITECTURE.md
   - Design API endpoint
   - Mock UI in frontend

2. **Backend Implementation**
   ```python
   # Add endpoint in app.py
   @app.route('/api/newfeature', methods=['GET'])
   @jwt_required()
   def new_feature():
       # Implementation
       pass
   ```

3. **Frontend Implementation**
   ```javascript
   // Add component in React
   const NewFeature = () => {
     const [data, setData] = useState(null);
     
     useEffect(() => {
       fetchData();
     }, []);
     
     return <div>{/* JSX */}</div>;
   };
   ```

4. **Testing**
   - Test locally
   - Test with multiple users
   - Test edge cases

5. **Deployment**
   - Push to repository
   - Run tests in CI/CD
   - Deploy to staging
   - Deploy to production

## 💡 Tips & Tricks

### Performance
- Use indexes on frequently queried fields
- Implement caching for leaderboards
- Lazy load components in React
- Minify CSS/JS in production

### Security
- Change all default passwords
- Use environment variables for secrets
- Enable HTTPS in production
- Regular security audits
- Keep dependencies updated

### Debugging
- Use `print()` in Python, `console.log()` in JS
- Check browser DevTools Network tab
- Monitor MongoDB queries with `db.setProfilingLevel(1)`
- Use Flask debug toolbar: `flask-debugtoolbar`

### Optimization
- Database query optimization
- Frontend bundle size analysis
- API response caching
- Lazy loading of questions
- Pagination for large datasets
