## 🚀 Quick Start (Choose One)

### Option 1: Local Development (Recommended for Development)
```bash
# Terminal 1 - Backend
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python app.py

# Terminal 2 - Frontend
cd frontend
npm install
npm start

# Terminal 3 - MongoDB (if not running)
mongod
```

### Option 2: Docker (Recommended for Production)
```bash
docker-compose up
```

Then visit: **http://localhost:3000**

## ✨ Features Implemented

### ✅ User Management
- User registration with email verification capability
- JWT-based authentication (30-day tokens)
- Password hashing with Werkzeug
- Profile viewing and statistics
- Role-based access (student/admin)

### ✅ House System
- 4 houses: Gryffindor, Hufflepuff, Ravenclaw, Slytherin
- Automatic random assignment on registration
- House-specific color themes
- House member tracking
- Collective house scoring

### ✅ Problem Management
- Create, read, update problems
- Difficulty levels: Easy, Medium, Hard
- Multiple test cases per problem
- Admin-only problem creation
- Time & memory limits configuration

### ✅ Code Execution & Judging
- Python code execution with test validation
- Configurable test cases (input/output)
- Real-time execution results
- Support for function-based problems
- Ready for C++ & Java (executor functions prepared)

### ✅ Scoring System
- Dynamic scoring based on:
  - Difficulty multiplier (1.0x, 1.5x, 2.5x)
  - Accuracy percentage (0-100%)
  - Speed bonus (up to 1.2x for fast submissions)
- House score aggregation
- Per-student average tracking

### ✅ Leaderboards
- **Global leaderboard**: Top 100 students
- **House leaderboard**: Ranking of 4 houses
- **House member leaderboard**: Rankings within a house
- Real-time updates
- Statistics display (problems solved, submissions, scores)

### ✅ User Dashboards
- Personal statistics (problems solved, average score)
- House standings view
- Quick action buttons
- Submission history

### ✅ Admin Panel
- Create new problems with rich forms
- Add multiple test cases
- Set difficulty and time limits
- View all submissions (ready to implement)
- Manage questions

### ✅ Responsive Design
- Works on desktop (1920px+)
- Tablet optimized (768px-1024px)
- Mobile friendly (below 768px)
- Beautiful gradient UI
- House color theming

## 💾 Database Schema

### Collections
- **users** (with indexes on email)
- **questions** (with indexes on difficulty)
- **submissions** (with indexes on user_id, question_id, timestamps)

### Documents Structure
Each collection has proper schema with fields for:
- User: stats, house assignment, role
- Questions: test cases, solution, difficulty
- Submissions: execution results, scores, timestamps
